二、推荐方案：修改hosts文件（最有效）
================================================================================

【方式1：使用自动配置脚本（推荐）】
--------------------------------------
1. 找到桌面上的文件：修改GitHub_Hosts.bat
2. 右键点击，选择"以管理员身份运行"
3. 按任意键确认
4. 等待配置完成
5. 关闭命令窗口

【方式2：手动修改hosts文件】
--------------------------------------
1. 以管理员身份打开记事本
   - 按Win键，搜索"记事本"
   - 右键点击，选择"以管理员身份运行"

2. 打开hosts文件
   - 点击"文件" → "打开"
   - 输入路径：C:\Windows\System32\drivers\etc\hosts
   - 文件类型选择"所有文件"
   - 点击"打开"

3. 在文件末尾添加以下内容
   - 打开桌面上的 github_hosts.txt
   - 复制所有内容
   - 粘贴到hosts文件末尾
   - 保存文件

4. 刷新DNS缓存
   - 按Win+R，输入cmd，按回车
   - 输入：ipconfig /flushdns
   - 按回车

【验证是否生效】
--------------------------------------
1. 打开浏览器
2. 访问：https://github.com
3. 如果能正常打开，说明配置成功
4. 访问：https://github.com/bzhanupsangejin/ai-vent-share
5. 检查是否能看到仓库内容

================================================================================
五、常见问题
================================================================================

【问题1】修改hosts后仍然无法访问
--------------------------------------
解决方案：
1. 确认hosts文件已保存
2. 刷新DNS缓存：ipconfig /flushdns
3. 重启浏览器
4. 清除浏览器缓存
5. 尝试使用无痕模式

【问题2】hosts文件无法保存
--------------------------------------
解决方案：
1. 确认以管理员身份运行记事本
2. 检查hosts文件是否被杀毒软件保护
3. 临时关闭杀毒软件
4. 使用自动配置脚本

【问题3】GitHub Desktop无法登录
--------------------------------------
解决方案：
1. 先修改hosts文件
2. 刷新DNS缓存
3. 重启GitHub Desktop
4. 使用浏览器登录GitHub
5. 在GitHub Desktop中点击"Sign in"

【问题4】克隆仓库失败
--------------------------------------
解决方案：
1. 检查网络连接
2. 确认hosts配置正确
3. 使用HTTPS而不是SSH
4. 尝试使用Gitee导入

【问题5】推送代码失败
--------------------------------------
解决方案：
1. 检查GitHub Token是否有效
2. 确认有仓库写权限
3. 检查网络连接
4. 使用GitHub Desktop推送
