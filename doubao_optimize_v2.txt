- 纯静态（GitHub Pages）
- 311条资源，8种分类
- 严格匿名（无ID）
- 分片索引（按类型）
优化建议？简要说3-5点核心的。
核心优化建议（5 点）
前端轻量化检索：基于纯 JS 实现「分类 + 关键词 + 模糊匹配」三合一检索，无需后端，检索结果即时渲染，适配 300 + 条资源的快速筛选需求。
静态资源性能调优：将分片索引 JSON 压缩（gzip）、资源列表按需懒加载，移除冗余 CSS/JS，降低 GitHub Pages 首屏加载体积，提升访问速度。
匿名体验极致化：禁用 GitHub Pages 默认统计、不写入任何本地存储（localStorage/cookie），资源跳转采用无追踪链接，彻底规避匿名性漏洞。
资源时效性维护：为每条资源添加「更新时间戳」，新增按 “最近更新” 排序维度，同时维护分类级别的失效资源清单（静态 markdown/JSON），保证 311 条资源的可用性。
移动端适配强化：优化分类导航的移动端交互（折叠式菜单）、检索框固定置顶，资源卡片自适应屏幕，解决静态页面在移动端的浏览效率问题。
总结
检索体验：纯前端实现多维度快速筛选，适配资源量级；
性能隐私：压缩静态资源 + 彻底规避匿名性风险；
实用价值：通过时间戳和失效清单保障资源时效性。
