Claude Skills Marketplace（https://skillsmp.com/zh）分析与推荐下载清单
一、平台核心优缺点
优点
技能资源海量且分类清晰：汇集超 10000 个开源 Claude Skills，按「开发、数据 & AI、工具、文档」等 12 个类别划分，支持关键词 / AI 语义搜索，快速定位需求
跨工具兼容性强：2025 年底 Agent Skills 成为开放标准，OpenAI（Codex CLI、ChatGPT）与 Claude 通用，安装一次可多平台使用
轻量化无负担：技能按需加载，扫描元数据仅需～100 tokens，完整加载约 5k tokens，不拖慢 AI 运行速度
质量基础过滤：筛选 GitHub 星星≥2 的仓库，剔除低质量技能，官方技能（Anthropic 出品）标注「Official」徽章，可信度高
安装与文档完善：每个技能提供详细安装指令（个人 / 项目 / 插件 3 种部署方式）、GitHub 仓库链接及使用场景说明，新手易上手
缺点
安全风险需警惕：社区技能为第三方开发，虽过滤低星仓库，但仍可能含恶意代码，需手动审查代码后安装
非官方技能质量不均：部分社区技能文档简陋、功能单一，甚至存在兼容性问题（如旧版 Claude 不支持）
无官方直接支持：平台为独立社区项目，与 Anthropic 无关联，遇到技能故障需联系原仓库开发者，无统一客服
更新存在延迟：技能更新依赖 GitHub 同步，仓库代码修改后需等待平台爬虫抓取（通常 1-2 天），无法实时获取最新版本

三、使用注意事项
安装前必做：复制 URL 后，先克隆 GitHub 仓库，打开「SKILL.md」确认功能描述，审查脚本代码（尤其含可执行命令的技能），避免恶意操作
部署路径规范：个人使用放「~/.claude/skills/」，团队共享放项目根目录「.claude/skills/」，便于 Git 提交同步
版本兼容检查：技能文档通常标注支持的 Claude 版本（如 Claude Code 3.x），老旧版本需优先选择「legacy」分支的技能
故障回滚：安装新技能后若出现 AI 异常，删除对应技能文件夹（如「~/.claude/skills/frontend-design」），重启 Claude 即可恢复
